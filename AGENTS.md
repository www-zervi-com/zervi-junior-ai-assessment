# Assessment Conventions

**Applies to:** Anyone working in this repository during the Zervi IT-AI junior assessment.

## Before you write code

1. Read this file.
2. Read `README.md`.
3. Read `docs/TASKS.md`.
4. Run `pytest` to see the current state.

## Code conventions

- Use type hints where they add clarity.
- Keep functions small and focused.
- Do not commit secrets, API keys, or personal access tokens.
- Write or update tests for any behavior you change or add.
- Run the full test suite before declaring a task done.

## Git conventions

- Make small, focused commits.
- Use clear commit messages (e.g., `Fix case-insensitive skill lookup`).
- Do not rewrite published history.

## When you get stuck

- Read the code and tests first.
- Ask a clarifying question if you have been stuck for more than 15 minutes.
- Explain your thinking out loud when pairing.

## Deliverables

- Day 1: all existing tests pass.
- Day 2: a new skill + tests, plus a short demo.
